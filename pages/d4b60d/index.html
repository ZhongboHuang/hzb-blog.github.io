    # 内存泄漏

    1. 什么是内存泄漏

    内存泄漏也称为“存储渗漏”，是使用动态存储分配函数动态分配的空间，使用后未释放，导致一直占据该内存单元，知道程序结束，即内存泄漏。

    2. 哪些操作会造成内存泄漏

    • setTimeout 的第一个参数为字符串
    • 闭包（两个对象彼此引用且彼此保留）

    # JS 防抖和节流

    • 防抖
    1. 含义

    当持续触发事件时，一定时间段内没有再触发事件，事件处理函数才会执行一次，若设定事件到来前再次触发了时间，则重新开始计时

    1. 代码实现
    function debounce(func, wait) {
  let timeout = null;
  return function () {
    if (timeout !== null) clearTimeout(timeout)
    timeout = setTimeout(func, wait)
  }
}

    • 节流
    1. 含义

    当持续触发事件时，保证一定时间段内只执行一次事件处理函数

    1. 代码实现
    function throttle(func, delay) {
  let prev = Date.now();
  return function () {
    let now = Date.now()
    let context = this
    let args = arguments
    if (now - prev >= delay) {
      func.apply(context, args)
      prev = now
    }  
  }
}

    # for...in 和 for...of 的区别

    • for...in 遍历的是数组的索引，即键名(key)；for...of 遍历的是数组的元素值(value)

    • for...of 是在循环调用遍历器接口(Symbol.iterator)

    # Object.defineProperty 和 Proxy 的区别

    1. Proxy 的优势
    • Proxy 直接监听整个对象而非属性
    • Proxy 可以直接监听数组的变化
    • Proxy 有多达 13 种拦截方法，是 Object.defineProperty 不具备的
    • Proxy 会返回一个对象，可以只操作新的对象达到目的，Object.defineProperty 只能遍历对象属性进行修改
    1. Object.defineProperty 的不足
    • Object.defineProperty 只能劫持对象的属性，所以必须对每个对象的每个属性进行遍历
    • Object.defineProperty 不能监听数组的变化，必须通过重写数组的7个操作方法来对数组进行监听
    • Object.defineProperty 无法对 es6 中的 Map 和 Set 对象的数据结构进行监听
    • Object.defineProperty 不能监听新增和删除操作
