# git 的基本指令

  1. 初始化
git config --global user.name "你的用户名" // 全局配置用户信息
git config --global user.email "你的邮箱"
git config --global --unset user.name // 删除配置的全局用户信息
git config --global --unset user.email 
git config --list

  1. 初始化仓库 git init hooks: 目录包含客户端或服务端的钩子脚本 info: 包含全局性排除文件 logs: 保存日志信息 objects: 目录存储所有数据内容 refs: 目录存储指向数据（分支）的提交对象的指针 config: 文件包含项目特有的配置选项 description: 用来显示对仓库的描述信息 HEAD: 文件指示目前被检出的分支 index: 文件保存暂存区信息

  2. git 操作最基本的流程 创建工作目录 对工作目录进行修改

    git add .
  git hash-object -w 文件名
  git update-index ...
git commit -m "提交的解释信息"
  git write-tree
  git commit-tree

  3. git 高层命令 CRUD

    • 初始化新仓库 git init
    • 检查当前文件状态 git status
    • 将所有修改文件加到暂存区 git add ./
    • 将暂存区的文件提交到版本库
      • git commit 进入编辑器编辑解释信息
      • git commit -m "解释信息"
      • 跳过使用暂存区域直接提交 git commit -a -m "解释信息" 需要处于被跟踪状态
    • 查看已暂存和未暂存的更新
      • 当前做的哪些更新还没有暂存 git diff
      • 有哪些更新已经暂存起来准备下次提交 git diff --staged
    • 查看提交的日志
      • git log
      • git log --oneline
    • git rm 文件名 将工作目录中的对应文件删除并将修改加到暂存区
    • git mv 源文件 重命名 将工作目录中的对应文件重命名并将修改加到暂存区

  4. 分支

    • 分支的本质是提交对象!!!
    • HEAD: 指向提交对象的指针, 默认指向 master, 切换分支就是让 HEAD 指向不同的提交对象 每次有新的提交时 HEAD 都会带着当前指向的分支一起向前移动
    • git branch 显示当前分支列表
    • 创建分支 git branch 分支名
    • 创建分支并切换 git checkout -b 分支名
    • 切换分支 git checkout 分支名
    • 强制删除分支 git branch -D 分支名 不能自己删自己
    • 删除已经被合并的分支或者空分支 git branch -d 分支名
    • 查看项目分叉历史 git log --oneline --decorate --graph --all
    • 查看每一个分支最后一次的提交对象 git branch -v
    • 新建分支并且使分支指向相应的提交对象 git branch 分支名 提交对象Hash
    • 合并分支 git merge 分支名
    • 查看已经合并到当前分支的分支列表 git branch --merged
    • 查看没有合并到当前分支的分支列表 git branch --no-merged

  5. 配置别名 git config --global alias.别名 命令 若命令为单个单词，不需要引号，否则需要加引号 例如：

    git config --global alias.co checkout 
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.det "log --oneline --decorate --graph --all"

  6. git 存储

    • git stash 将未完成的修改保存到一个栈上
    • git stash list 查看存储
    • git stash apply stash@{2} 应用指定的储藏，若不指定默认应用最近的储藏
    • git stash drop stash@{2} 移除指定的储藏
    • git stash pop 应用最近的储藏，并从栈中移除它

  7. 撤销

    • 撤回工作区的修改 git checkout -- <文件名>
    • 撤回暂存区的暂存 git reset HEAD <文件名>
    • 重新提交修改上一次的提交注释 git commit --amend

  8. 重置（三部曲）

    • 第一部:
      • git reset --soft HEAD~ 只动 HEAD, 但会带着分支一起移动, 不会修改工作目录和暂存区
      • git reflog 打印 HEAD 修改日志 git reset --soft HEAD <hash> 移动到指定 hash 的分支
    • 第二部:
      • git reset [--mixed] HEAD~ 动 HEAD, 同样带着分支一起移动, 还会将暂存区的东西回滚到上一次暂存
    • 第三部:
      • git reset --hard HEAD~ 撤销最后的 git add 和 git commit 命令以及工作目录的所有工作

  9. 打 tag

    • git tag <tagName> 给最近的一个提交对象打 tag
    • git tag <tagName> <commitHash> 指定提交对象打 tag
    • 问题：使用 git checkout <tagName> 会出现头部分离 解决方法：git checkout -b <分支名> 创建一个分支，让 HEAD 指向它

  10. 远程仓库

    • 配置远程别名 git remote add 别名 远程仓库地址
    • 配置用户信息
      git config user.name "huangzhongbo"
git config user.email "huangzhongbo@cmict.chinamobile.com"
      1
      2
    • 克隆仓库到本地 git clone 仓库地址

    只有在克隆仓库时，本地分支和远程跟踪分支有同步关系，可以直接 git push

    • 更新修改

      git fetch 别名（将修改同步到远程跟踪分支）
git merge 远程跟踪分支
      1
      2

    • 本地分支跟踪远程分支 git branch -u 远程跟踪分支

    • 公开分享一个分支，将其推送到远程仓库 git push 别名 分支名

    • 本地新建分支并制定远程跟踪分支 git branch -b 分支名 [remoteName]/[branch] 快捷方式 git branch --track [remoteName]/[branch]

    • 查看设置的所有跟踪分支 git branch -vv

    • 删除远程分支

      1. git push origin --delete 分支名 删除远程分支
      2. git remote prune origin --dry-run 列出仍在远程跟踪，但是远程已被删除的无用分支
      3. git remote prune origin 清除上面命令列出来的远程分支

    • 下载指定远程仓库分支 git clone -b 分支名 ssh地址

  11. pull request

    • 当已经 fork 了一次，需要重新提 pull request
      1. git remote add <shortname 源仓库> <url 源仓库地址>
      2. git fetch 远程仓库名字
      3. git branch -u 远程跟踪分支

  12. git 的核心概念

    1. 三个区域：工作目录、暂存区、版本库
    2. 三个对象：git对象、tree对象、commit对象
    3. 三种分支：本地分支、远程追踪分支、远程分支
