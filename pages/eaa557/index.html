    # 从浏览器地址栏输入 url 到请求返回都发生了什么？

    1. url 解析（编码）
    2. dns 解析
    3. tcp 连接（http 三次握手）
    4. http 请求 html
    5. 请求返回
    6. 浏览器页面渲染
    7. http 四次挥手断开连接

    # DNS 解析流程，html 如何做 dns 优化？

    1. 比如查询 www.baidu.com 域名，首先查询本地的 dns 查询记录，也就是 hosts 文件
    2. 如果没有，查询本地 dns 服务器
    3. 仍没有之后，有两种模式
      • 迭代查询：依次查询根服务器、.com 域服务器、.baidu.com 域服务器
      • 递归查询：只需要发送一起请求，转发器就会将最终结果返回给本地 dns 服务器

    优化：可以在html页面头部加上dns缓存

    <meta http-equiv="x-dns-prefetch-control" content="on" />
<link rel="dns-prefetch" href="http://bdimg.share.baidu.com" />

    # http 三次握手，两次为什么不行？三次握手都发生了什么？

    1. 客户端发送一个 TCP 的 SYN=1, Seq=x 的包到服务器
    2. 服务端返回 SYN=1, ACK=X+1, Seq=y 的响应包
    3. 客户端发送 ACK=y+1, Seq=z

    两次的话无法确定客户端是否成功收到响应包。

    # 浏览器如何解析 html？

    1. 构建 DOM (Document Object Model)
    2. 构建 CSSOM (CSS Object Model)
    3. 执行 JavaScript
    4. 构建 Render Tree (DOM + CSSOM)
    5. 布局 (Layout)
    6. 绘制 (Print)

    # 一个页面上有大量的图片，加载很慢，有哪些方法可以优化图片的加载？

    • 使用 Sprites 图片：将多个零星图片放到一张图片，减少 http 请求次数
    • 压缩图片
      • 展示区域小于图片真实大小，可在服务端压缩至与展示一致
    • 延迟可视区域外的图片显示（图片懒加载）
      • 若为相册类，可设计为当前图片和其前一张和后一张优先加载
    • 图片格式优化
    • 使用 Progressive JPEGs: 这种图片是 JPEG 格式的一个特殊变种，名为“高级 JPEG”。在创建高级 JPEG 文件时，数据是这样安排的：在装入图像时，开始只显示一个模糊的图像，随着数据的装入，图像逐步变得清晰。它相当于交织的 GIF。

    # 请描述下 cookies, sessionStorage 和 localStorage 的区别

    • cookies 一般用来储存用户登录标识，即使不需要，也会在同源的 http 请求中携带

    • 存储大小：cookies 的大小不能超过 4k，sessionStorage 和 localStorage 的大小没有限制，可以达到 5M 或更大

    • 存储时间：localStorage 不主动清除会一直保存在浏览器中；sessionStorage 会随着浏览器的关闭被清除；cookies 在设置过期时间前一直有效。

    # 为什么利用多个域名来存储网站资源会更有效？

    • CDN 缓存更方便
    • 突破浏览器并发限制
    • 节约 cookies 带宽
    • 节约主域名的连接数，优化页面响应速度
    • 防止不必要的安全问题

    # 渐进增强和优雅降级之间的不同

    • 渐进增强：针对低版本浏览器进行构建页面，保证其基本功能；再针对高级浏览器进行交互改进和追加功能以达到更好的用户体验

    • 优雅降级：一开始就构建完整功能，然后对低版本浏览器进行兼容

    # 对于 HTML 语义化的理解

    • 用正确的标签做正确的事
    • 语义化让页面的内容结构化、结构更清晰，便于对浏览器、搜索引擎解析；即使在没有css样式情况下也以一种文档格式显示
    • 搜索引擎的爬虫也依赖于 HTML 标记来确定上下文和各个关键字的权重，利于 SEO
    • 使阅读源码的人对网站更容易分块，便于阅读理解

    # iframe 框架的优缺点

    1. 优点

    • 可以原封不动的把嵌入的网页展示出来
    • 若存在多个网页应用 iframe，修改时只需修改 iframe 内容
    • 若遇到加载缓慢的第三方内容（图标或广告），可以用 iframe 解决

    2. 缺点

    • 会出现各种滚动条
    • 会阻塞主页面的 OnLoad 事件
    • 搜索引擎的检索程序无法解读这种页面，不利于 SEO
    • 与主页面共享连接池。浏览器对相同域的连接有限制，影响页面的并行加载

    # src 和 href 的区别

    • src：标识引用资源，替换当前元素，用在 img, script, iframe 上，是页面不可或缺的一部分。当浏览器解析到 src 时，会暂停其他资源的下载和处理（图片除外），知道该资源加载、编译、执行完毕
    • href：标识超文本引用，用在 link 和 a 上，是在当前元素和引用资源之间建立联系

    # 前端页面由哪三层组成？

    结构层(structural layer)、表示层(presentation layer)、行为层(behavior layer)

    # 网页验证码是干嘛的？可以解决什么安全问题？

    区分用户是计算机还是人的公共全自动程序。可以防止恶意破解密码，刷票，论坛灌水，有效防止黑客对某一个特定注册用户用特定程序暴力破解方式进行不断登录尝试。
