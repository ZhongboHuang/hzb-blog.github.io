    # BFC

    1. 什么是 BFC？

    BFC - Block Formatting Context 块级格式化上下文；它是一块块独立的渲染区域，不会影响到外部元素，包含创建该 BFC 区域的上下文元素下的所有子元素，但不包括创建了新的 BFC 的子元素的子元素。

    <div class="box1" id="bfc1">
  <div class="box2"></div>
  <div class="box3"></div>
  <div class="box4"></div>
  <div class="box5" id="bgc2">
    <div class="box6"></div>
    <div class="box7"></div>
    <div class="box8"></div>
  </div>
</div>

    假设 box1 和 box5 是两个 bfc 区域，则 box1 包括 box2, box3, box4, box5, 但不包括 box6, box7, box8；box5 包括 box6, box7, box8。

    2. BFC 的原理

    • BFC 元素垂直方向的外边距会重叠，不同 BFC 外边距不会重叠。
    • BFC 区域不会与浮动元素的布局重叠。
    • BFC 是一个独立的容器，外面的元素不会影响里面的元素，里面的元素不会影响外面的元素。
    • 计算 BFC 高度时，浮动元素也会计算在内(清除浮动)。

    3. 如何创建 BFC?

    • body 根元素
    • float 值不为 none
    • position 的值不为 static 或 relative
    • display 的值为 inline-block, table, table-cell, flex, inline-flex

    4. 可以解决那些布局问题

    • 外边距塌陷
    • 包含塌陷（设置子元素外边距，父元素也跟着一起动）
    • 清除浮动
    • 阻止标准流元素被浮动元素覆盖

    # display: none; 和 visibility: hidden; 的区别

    # rgba() 和 opacity 的透明效果有什么不同

    opacity 作用于整个元素，以及元素内的所有内容的透明度，会被子元素继承

    rgba() 只作用于元素的颜色或背景色，不会被继承

    # CSS 的盒子模型

    1. 分类：标准盒模型和怪异盒模型
    2. 区别：
      • 标准盒模型的 width 和 height 指得是 content 的大小；盒子大小为 content + padding + border + margin
      • 怪异盒模型 box-sizing: border-box : width 和 height 指得是内容、内边距、边框的和 (content + padding + border)；盒子大小为 (width + margin)

    # 图片设置 heigh 为 100% 纵向出现了滚动条

    1. 原因

    父元素为 inline-block 时，行内元素默认的 vertical-align 值为 baseline，所以其对齐的基线并不是父元素的边缘，而是其下方的空白元素，所以实际高度超过了父元素的实际高度，导致出现了纵向滚动条

    1. 解决办法
    /* 1. 修改子元素图片基线对齐方式 */
img {
  vertical-align: bottom
}

/* 2. 对父元素使用 overflow-y: hidden */
/* 3. 对父元素使用line-height: 0; */
/* 4. 对父元素使用font-size: 0; */
/* ... */

    # link 和 @import 的区别

    • link 是 HTML 方式，@import 是 css 方式
    • link 最大限度支持串行下载，@import 过多嵌套导致串行下载，会出现 FUNC (文档样式缺失)
    • link 可以通过 rel="alternate stylesheet" 指定候选样式
    • @import 必须在样式规则之前，可以在 css 文件中引入其它文件
    • 浏览器对于 link 的支持要早于 @import，可以使用 @import 对老浏览器隐藏样式
